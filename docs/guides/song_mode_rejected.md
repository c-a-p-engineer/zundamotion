# 歌唱機能 (`song`) 不採用メモ

- 記録日: 2026-05-24
- 状態: 不採用
- 対象: Zundamotion 本体への歌唱機能追加

関連:

- [docs 入口](../README.md)
- [未確定の課題](../issues_pending.md)
- [README](../../README.md)

## 背景

VOICEVOX song API を利用して、`items.song` から歌唱 WAV を生成し、既存の timeline / 口パク / 動画合成パイプラインへ流す案を試作した。  
技術的には成立する見込みがあったが、Zundamotion の利用目的と保守範囲に照らして採用可否を再評価した。

## 不採用理由

1. 字幕との関係で、長い歌を歌わせると字幕 cue が長くなりやすい。
2. 長時間歌唱を YAML の `notes` や `melody + lyrics` で管理するコストが高い。
3. 音楽や既存音源と組み合わせる場合、同期・責務分離・仕様整理の複雑度が大きく増える。
4. Zundamotion は動画作成ツールであり、歌唱機能を本体機能として抱えると将来コストが高い。

## 代替方針

- 歌や長尺の音声は別途作成した音声ファイルを用意する。
- Zundamotion 側では既存の BGM / 外部音声取り込みで扱う。
- 必要なら動画側の字幕や演出だけを Zundamotion で担当する。

## 再検討する場合の前提条件

- 1 item 内で phrase 単位に字幕を自然に分割できること。
- 長時間歌唱を低コストで書ける記法があること。
- BGM / 楽曲 / 外部音声との責務分離が明確であること。
- 本体の保守範囲として持つ価値があることを説明できること。

## 補足

今回の判断は「歌唱 API が使えない」という意味ではなく、「Zundamotion 本体機能としては持たない」という判断である。  
将来、用途を限定した別設計や外部ツール連携として再検討する余地はある。
